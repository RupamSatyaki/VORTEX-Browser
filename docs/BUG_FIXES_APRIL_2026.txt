ROOT CAUSE ANALYSIS
================================================================================

PRIMARY BUG (Cascade Failure):
  Error #2 — storage:read duplicate handler registration
  
  Flow:
    main.js line 162: Storage.registerStorageHandlers()
      → registers storage:read, storage:write, storage:delete, storage:dir
    
    main.js line 164: IpcHandler.registerHandlers()
      → calls storageHandlers.register()
      → tries to register storage:read AGAIN
      → Electron throws error and CRASHES handler registration
      → All subsequent handlers (passwords, addresses, app, etc.) never load
  
  Impact: Errors #3-8 are cascade failures from this crash

SECONDARY BUG:
  Error #1 — WhatsAppPanel module missing
  
  Flow:
    navigation/index.js line 33: WhatsAppPanel.toggle()
    But no whatsappPanel.js file exists
  
  Impact: Toolbar WhatsApp button throws error on click

  Error #9 — dragEvent is not defined
  
  Flow:
    Electron internal webview drag implementation bug
    Not from our code — tabDrag.js uses _drag variable correctly
  
  Impact: Console noise only — no functional issue

FIX #1: Remove duplicate storage handler registration

  Storage.registerStorageHandlers();
  // Ensure default storage files exist on first run

Reason: IpcHandler.registerHandlers() already calls storageHandlers.register()
        Duplicate call caused Electron crash


FIX #2: Add callback support to storageHandlers
────────────────────────────────────────────────────────────────────────────────
File: vortex/src/main/ipc/storageHandlers.js

ADDED:
  // Callbacks to invoke when specific files are written
  const _writeCallbacks = {};

  function onWrite(name, fn) {
    _writeCallbacks[name] = fn;
  }

MODIFIED storage:write handler:
  ipcMain.handle('storage:write', (_e, name, data) => {
    const result = writeFile(name, data);
    if (_writeCallbacks[name]) _writeCallbacks[name](data);  // ← NEW
    return result;
  });

ADDED storage:dir handler:
  ipcMain.handle('storage:dir', () => STORAGE_DIR);

EXPORTED:
  module.exports = { register, onWrite };

Reason: Allows downloadHandlers to hook into settings writes without
        overriding the handler (previous hacky approach)


FIX #3: Remove hacky storage:write override
────────────────────────────────────────────────────────────────────────────────
File: vortex/src/main/ipcHandler.js

BEFORE:
  require('./ipc/storageHandlers').register();
  require('./ipc/appHandlers').register(_getWin);
  require('./ipc/windowHandlers').register(_getWin);
  require('./ipc/miscHandlers').register(_getWin);

  const dlHandlers = require('./ipc/downloadHandlers');
  dlHandlers.register(pushToRenderer);

  // Override storage:write to refresh download settings when settings.json saved
  ipcMain.removeHandler('storage:write');
  ipcMain.handle('storage:write', (_e, name, data) => {
    const { writeFile } = require('./storage');
    const result = writeFile(name, data);
    if (name === 'settings') dlHandlers.refreshDlSettings();
    return result;
  });

AFTER:
  const storageHandlers = require('./ipc/storageHandlers');
  const dlHandlers = require('./ipc/downloadHandlers');

  // Register storage handlers first; hook settings writes to refresh dl settings
  storageHandlers.register();
  storageHandlers.onWrite('settings', () => dlHandlers.refreshDlSettings());

  require('./ipc/appHandlers').register(_getWin);
  require('./ipc/windowHandlers').register(_getWin);
  require('./ipc/miscHandlers').register(_getWin);

  dlHandlers.register(pushToRenderer);

Reason: Clean callback approach instead of removeHandler + re-register hack
