================================================================================
OLLAMA INTEGRATION
================================================================================

Base URL    : http://localhost:11434 (settings mein configurable)
Models API  : GET /api/tags
Chat API    : POST /api/chat (streaming)
Stream      : ReadableStream token-by-token
Abort       : AbortController (stop generation button)

Request Format:
  {
    model: "llama3:8b",
    messages: [...],
    stream: true,
    options: { temperature: 0.7 }
  }

Error Handling:
  - Ollama not running → "Ollama not found. Start with: ollama serve"
  - No models → "No models found. Run: ollama pull llama3"
  - Network error → retry button

================================================================================
THINKING MODEL SUPPORT
================================================================================

Models: deepseek-r1, qwen3, phi4-reasoning, etc.

Stream Parse Flow:
  Token aaya
      ↓
  <think> detect? → thinking buffer start, show spinner "Thinking..."
      ↓
  </think> detect? → thinking done, show "Thought for Xs" accordion (collapsed)
      ↓
  Normal content → chat bubble mein stream karo

UI States:
  1. Thinking (streaming) : 🔄 spinner + "Thinking..." (collapsed, not expandable yet)
  2. Thinking (done)      : 🤔 "Thought for 3s" ▶ (click to expand/collapse)
  3. Content (streaming)  : text appears token by token
  4. Content (done)       : full message, copy button appears

================================================================================
PAGE READING
================================================================================

pageReader.js:
  - webview.executeJavaScript() se document.body.innerText extract
  - Script/style/nav/footer elements remove
  - 8000 char limit (LLM context budget)
  - Returns: { url, title, text, selectedText }

Auto-trigger keywords:
  "is page", "this page", "current page", "summarize", "explain this",
  "what does this say", "translate this" → auto read_page call

Manual trigger:
  📎 button in input bar → attach page content to next message

================================================================================
CONTEXT BUILDER
================================================================================

contextBuilder.js assembles:
  1. System prompt (tool schemas + instructions)
  2. Page context (if read_page called or auto-triggered)
  3. Chat history (last 20 messages)
  4. Current user message

System Prompt includes:
  - Browser assistant role description
  - All 34 tool schemas (auto-generated from mcpRegistry)
  - Tool call format: {"tool": "tool_name", "args": {...}}
  - Response language: match user's language
  - Thinking model instructions

================================================================================
STREAM PARSER — STATE MACHINE
================================================================================

States: IDLE → THINKING → CONTENT → TOOL_CALL

  IDLE:
    <think> found → switch to THINKING
    { found → switch to TOOL_CALL
    else → switch to CONTENT

  THINKING:
    </think> found → switch to CONTENT, fire onThinkingDone(text, seconds)
    else → fire onThinkToken(token)

  CONTENT:
    {"tool": found → switch to TOOL_CALL
    else → fire onContentToken(token)

  TOOL_CALL:
    complete JSON → fire onToolCall(name, args), switch to CONTENT

================================================================================
TOOL FILE FORMAT (Standard Pattern)
================================================================================

Every tool file follows this exact pattern:

  // mcp/tools/tabs/createTab.js
  const createTab = {
    name: 'create_tab',
    description: 'Open a new browser tab with the given URL',
    parameters: {
      url: {
        type: 'string',
        description: 'Full URL to open (e.g. https://google.com)',
        required: true
      },
      background: {
        type: 'boolean',
        description: 'Open tab in background without switching focus',
        required: false,
        default: false
      }
    },
    execute({ url, background = false }) {
      try {
        // implementation using global Tabs, WebView, vortexAPI etc.
        return { success: true, result: { tabId: '...' } };
      } catch (err) {
        return { success: false, error: err.message };
      }
    }
  };

NOTES & CONSTRAINTS
================================================================================

1. Ollama must be running locally (ollama serve)
2. webviewPreload.js already handles page content access
3. Existing Tabs, WebView, Navigation globals use karo — no new IPC needed for most tools
4. vortexAPI.invoke() use karo IPC calls ke liye (screenshot, bookmark, etc.)
5. Panel state persist karo (open/closed) across tab switches
6. Assistant panel sirf main window mein hoga (incognito mein nahi)
7. No external API calls — fully local Ollama only
8. Tool calls ek message mein multiple ho sakte hain (chain tools)
