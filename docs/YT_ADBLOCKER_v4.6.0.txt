APPROACH: Dedicated Electron Session + Auto Skip + Smart Pattern Matching
STATUS: ✅ Working — No Detection

================================================================================
HOW IT WORKS
================================================================================

YouTube runs in a DEDICATED Electron session (persist:youtube) that has its own
webRequest interceptor. This is completely separate from the main browser session.

Flow:
  1. User navigates to youtube.com
  2. Webview is created with partition='persist:youtube'
  3. YouTube session has webRequest.onBeforeRequest registered
  4. Ad video requests are blocked at network level
  5. Skip button is auto-clicked via executeJavaScript
  6. Overlay/companion ads are hidden via display:none

================================================================================
WHY THIS WORKS (NO DETECTION)
================================================================================

Previous approaches that FAILED:
  ❌ Blocking imasdk.googleapis.com → YouTube detects IMA SDK failure
  ❌ DOM removal (el.remove()) → YouTube's MutationObserver detects missing elements
  ❌ display:none on ad containers → YouTube DOM integrity check detects it
  ❌ playbackRate = 16 → YouTube monitors video.playbackRate
  ❌ video.currentTime jump → YouTube monitors currentTime changes
  ❌ fetch/XHR override → YouTube checks native function toString()
  ❌ window.__vxObs global → YouTube scans for unknown globals (fingerprint)
  ❌ Blocking /youtubei/v1/log_event → YouTube uses this for ALL events, not just ads
  ❌ Blocking /pagead/id → YouTube JS catches ERR_BLOCKED_BY_CLIENT and detects adblock
  ❌ data: URL redirect for pagead/id → Electron blocks with ERR_UNSAFE_REDIRECT
  ❌ MutationObserver on document.documentElement → Too aggressive, triggers integrity checks

Current approach that WORKS:
  ✅ Session-level webRequest blocking → YouTube sees "network error", not adblock
  ✅ IMA SDK loads normally → No detection trigger
  ✅ Only ad VIDEO CONTENT URLs blocked (ctier=L and other ad params)
  ✅ /youtubei/v1/log_event ALLOWED → YouTube's event bus must not be blocked
  ✅ /pagead/id ALLOWED → Ad ID endpoint must not be blocked
  ✅ /api/stats/atr blocked ONLY when el=adunit or is_ad=1 (not el=detailpage)
  ✅ Skip button click → YouTube expects this, not suspicious
  ✅ Randomized non-enumerable key instead of window.__vxObs → No fingerprint
  ✅ MutationObserver on #movie_player only → Less suspicious than full DOM watch
  ✅ setInterval(skipAd, 1000) fallback → Catches ads that slip through observer

AD PATTERNS BLOCKED (adPatterns.js)
================================================================================

1. Ad video content (googlevideo.com):
   - ctier=L        → Ad content tier (classic signal)
   - ctier=A&       → Some ad variants
   - oad=1          → Original ad flag
   - &oad=          → Alternate oad position
   - adformat=      → Ad format parameter
   - ad_type=       → Ad type parameter
   - adunit=        → Ad unit identifier
   - ad_len=        → Ad length parameter (newer)
   - adslot=        → Ad slot parameter
   - correlator=    → Ad correlator
   - adsid=         → Ad session ID
   - adtagurl=      → Ad tag URL
   - pucrd=         → Ad-specific param (newer YouTube)
   - rdid=          → Rewarded ad ID
   - vad_type=      → Video ad type
   - idpj=          → Ad impression param
   - label=preroll  → Preroll label
   - label=midroll  → Midroll label
   - label=postroll → Postroll label

2. YouTube ad tracking endpoints:
   - /api/stats/ads
   - /pagead/
   - /ptracking
   - /get_midroll_info
   - /ad_data_204
   - /api/stats/qoe?adformat
   - /pagead/adview
   - /pagead/conversion/
   - /pagead/viewthroughconversion/
   - /api/stats/atr (ONLY when el=adunit OR is_ad=1 — NOT el=detailpage)

3. Ad domain paths (always blocked regardless of referrer):
   - /pagead/adview
   - /pagead/conversion
   - /simgad/             → tpc.googlesyndication.com image ads
   - /safeframe/          → Safe frame ad container
   - /activeview/         → Active view tracking
   - /xbbe               → Ad beacon
   - /ddm/activity        → DoubleClick activity

4. Ad domains (blocked when referrer is youtube.com):
   - pagead2.googlesyndication.com
   - tpc.googlesyndication.com
   - ad.doubleclick.net
   - googleads.g.doubleclick.net
   - pubads.g.doubleclick.net
   - securepubads.g.doubleclick.net
   - googleadservices.com

NOT BLOCKED — DETECTION TRIGGERS (never block these):
   - imasdk.googleapis.com          → IMA SDK must load
   - /youtubei/v1/log_event         → General event bus (video play, navigation, etc.)
   - /pagead/id                     → Ad ID fingerprinting endpoint
   - /api/stats/atr?el=detailpage   → Regular video stats
   - /api/timedtext                 → Subtitles/captions
   - /youtubei/v1/player            → Player initialization
   - fonts.googleapis.com           → Google Fonts (blocked by EasyPrivacy — whitelisted)

DOM MANIPULATION (ytAdblock.js — executeJavaScript)
================================================================================

Injected via wv.executeJavaScript() on dom-ready and did-navigate.

Anti-fingerprint measures:
  - No window.__vxObs or any known global name
  - Uses randomized non-enumerable document property (_vx + random string)
  - Object.defineProperty with enumerable:false → invisible to Object.keys()
  - MutationObserver targets #movie_player only (not document.documentElement)
  - setInterval(skipAd, 1000) as fallback

1. Skip button auto-click (tries all known selectors):
   .ytp-ad-skip-button-modern
   .ytp-ad-skip-button
   .ytp-skip-ad-button
   .videoAdUiSkipButton
   [class*="skip-button"]
   button.ytp-ad-skip-button-container button

2. Overlay banner hide (style.setProperty with !important):
   .ytp-ad-overlay-slot    ← Bottom overlay banner

3. Companion/sponsored card hide:
   #companion-ad-container
   ytd-companion-slot-renderer
   #google-container-id
   .ytp-ad-overlay-container
   .ytp-image-background-gradient-vertical

================================================================================
TROUBLESHOOTING — DETECTION WAPAS AAYE TO
================================================================================

STEP 1 — DevTools Network tab kholo
  YouTube tab → Ctrl+Shift+I → Network → filter:
  doubleclick OR googlevideo OR youtube.com/api OR youtube.com/youtubei

STEP 2 — Kya dekho

  Yeh ERR_BLOCKED_BY_CLIENT hona chahiye (good):
    ctier=L googlevideo URLs
    pagead/adview
    api/stats/ads
    simgad/
    ddm/trackimp

  Yeh kabhi block NAHI hona chahiye (detection triggers):
    /youtubei/v1/log_event   → general event logging
    /youtubei/v1/player      → player init
    /api/timedtext           → subtitles
    fonts.googleapis.com     → fonts
    pagead/id                → ad ID
    imasdk.googleapis.com    → IMA SDK (KABHI BLOCK MAT KARO)

STEP 3 — Fix kahan karna hai

  | Problem                              | File              | Fix                          |
  | Naya ad URL block nahi ho raha       | adPatterns.js     | AD_VIDEO_PARAMS mein add karo|
  | Zaruri URL galti se block ho raha    | adPatterns.js     | YT_SAFE_PATHS mein add karo  |
  | Blocklist se site galti se block     | blocklist/engine  | _WHITELIST mein add karo     |
  | DOM injection detect ho rahi hai     | ytAdblock.js      | Skip selectors update karo   |

STEP 4 — Quick checklist

  1. Network log mein dekho kaunsa URL ERR_BLOCKED_BY_CLIENT hai
  2. Agar woh "safe" URL hai → YT_SAFE_PATHS mein add karo
  3. Agar naya ad URL hai → AD_VIDEO_PARAMS mein add karo
  4. App restart karo (main process changes ke liye)
  5. YouTube tab close karke naya tab kholo (partition reset)

GOLDEN RULES — Kabhi mat bhoolna:
  ✅ ctier=L → hamesha block
  ✅ imasdk.googleapis.com → hamesha ALLOW (detection trigger)
  ✅ log_event → hamesha ALLOW (detection trigger)
  ✅ pagead/id → hamesha ALLOW (detection trigger)
  ✅ fonts/subtitles/player → hamesha ALLOW
  ✅ App restart zaruri hai main process changes ke baad
  ✅ YouTube new tab mein kholna padega partition reset ke liye

================================================================================
KNOWN LIMITATIONS
================================================================================

1. YouTube must be opened in a NEW TAB after enabling for session to take effect
   (Electron partition cannot be changed after webview creation)

2. Some unskippable ads may still show briefly before skip button appears
   (YouTube controls when skip button becomes clickable)

3. Homepage banner ads may occasionally flash before being hidden
   (display:none applied after DOM mutation, not before render)

4. YouTube periodically updates its detection algorithms — patterns may need
   updating. Use the troubleshooting guide above when detection occurs.
